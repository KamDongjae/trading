# Pyramide — 빗썸/바이낸스 기반 모의투자(페이퍼 트레이딩) 시스템

실제 돈은 안 쓰고, 빗썸(원화 시세) + 바이낸스 선물(USD 시세·지표) 데이터를 실시간으로
받아와서 롱/숏 포지션을 가상으로 굴려보는 모의투자 시스템입니다. 서버(계좌·점수 계산)와
클라이언트(GUI)가 분리되어 있어서, 서버는 Termux에서 24시간 계속 돌리고 클라이언트만
껐다 켰다 해도 포지션은 서버가 계속 감시합니다.

---

## 1. 구성 파일

| 파일 | 역할 |
|---|---|
| `trading_server.py` | 계좌/포지션 엔진. 시세·지표·점수 계산, 강제청산 감시, 명령 처리. **GUI 없음, 헤드리스로 돌아감.** |
| `trading_client.py` | 주문용 GUI (Tkinter). 코인 목록/포지션 목록을 보고 진입·청산·자금 관리. |
| `data_collector.py` | 서버와 별개로, 시세·지표·점수를 CSV로 계속 로깅만 하는 수집기. 나중에 오프라인 분석용. |
| `screener_prepump.py` | `data_collector.py`가 쌓은 CSV를 오프라인에서 매집/분산 스코어로 재계산·스크리닝하는 독립 스크립트. |
| `requirements.txt` | pip 의존성 목록. |

**중요**: `data_collector.py`는 `trading_server.py`를 모듈로 import해서 지표 계산 함수를 그대로 재사용합니다. 데이터 수집 기기에도 `trading_server.py` 파일을 같이 둬야 합니다(실행은 안 됨, import만 함).

## 2. 설치

### 2-1. 서버 기기 (trading_server.py, data_collector.py)

```bash
pip install -r requirements.txt
```

Termux라면 pandas/numpy는 pip보다 pkg로 먼저 까는 걸 추천합니다(컴파일 시간 단축):
```bash
pkg install python-numpy tur-repo
pkg update
pkg install python-pandas
pip install requests dnspython pybithumb python-dateutil pytz tzdata six fpdf2
```

### 2-2. 클라이언트 기기 (trading_client.py)

pip로 설치할 게 없습니다. tkinter만 있으면 됩니다.
- Termux: `pkg install x11-repo && pkg install python-tkinter` (메인 저장소가 아니라 **x11-repo**에 있음. 화면 렌더링에는 Termux:X11이나 VNC 같은 디스플레이 서버가 별도로 필요)
- proot-distro Ubuntu 안: `apt install python3-tk`
- 한글이 깨지면: `apt install fonts-nanum fonts-noto-cjk locales fontconfig && fc-cache -fv` + 로케일 설정(`locale-gen ko_KR.UTF-8`)

## 3. 실행 순서

```bash
# 1) 서버 먼저 (같은 기기든 다른 기기든 상관없음, 24시간 켜두는 걸 권장)
python trading_server.py

# 2) 클라이언트 (서버가 쓴 CSV를 읽기만 함)
python trading_client.py
```

서버와 클라이언트는 **CSV 파일과 명령 폴더**로 통신합니다(소켓/포트 없음):
- `server_market.csv` — 서버 → 클라이언트, 코인별 시세/지표/점수 스냅샷
- `server_account.csv` — 서버 → 클라이언트, 잔고/포지션 스냅샷
- `server_cmds/` — 클라이언트 → 서버, 주문/설정 명령 큐

기본 저장 위치는 안드로이드 공용 저장소(`/storage/emulated/0/Documents`, `termux-setup-storage` 필요)이고, 접근 불가 시 스크립트 폴더로 자동 대체됩니다. **서버·클라이언트가 서로 다른 기기/앱이면 같은 폴더를 보고 있는지 반드시 확인하세요** — 둘 다 저장공간 권한이 있어야 합니다.

## 4. 점수 체계 (v3, 2026-07 기준)

### 4-1. long_score / short_score — 105점 만점 (추세추종형)

| 항목 | 배점 |
|---|---|
| EMA 삼중(20/60/120) 정배열 | 20 |
| 가격위치(RSI+BB% 결합조건) | 20 |
| CVD 추세 | 15 |
| OI(미체결약정) | 15 |
| VolZ(거래량) | 15 |
| 30분 모멘텀 | 15 |
| ATR/거래대금 유동성 필터 | 5 |

`raw합/105×100`으로 0~100점 환산. L/S 비율·Funding·RSI Delta는 v3에서 완전히 제외(노이즈 유발 판단). 진입 컷은 시장 상태(전 코인 평균 ATR%)에 따라 70(추세장)~80(횡보장) 자동 조정, 기본 75.

### 4-2. prepump_score / preshort_score — 100점 만점 (장기 매집/분산 사이클 탐지)

롱/숏 점수와 완전히 다른 로직입니다 — **EMA가 '정배열'이면 오히려 감점**(이미 매집이 끝나고 추세가 시작된 신호로 봄).

| 항목 | 배점 |
|---|---|
| OI 지속 증가(매집) / 감소·불일치(분산) | 25 |
| CVD 누적 증가(매집) / 지속 감소(분산) | 20 |
| EMA 압축도(매집: 겹칠수록 고득점) / 과이격(분산) | 15 |
| ATR | 10 |
| VolZ | 10 |
| 가격 위치(최근 20캔들 박스 내 위치) | 10 |
| RSI(박스권 중립대 우대) | 5 |
| 최근 상승률 패널티(매집) / 급등 보너스(분산) | 5 |

고정 컷 80점.

### 4-3. Predict Score — 100점 만점 (포지션 카드 전용, 진입 후 예측)

지금 수익(PNL)과 완전 별개로 "이 포지션 방향이 앞으로도 유지될지"를 예측합니다.

| 항목 | 배점 |
|---|---|
| 가격 기울기(Slope) | 30 |
| 가속도(Acceleration) | 25 |
| CVD 방향일치 | 20 |
| EMA 방향일치 | 15 |
| Level(진입 당시 원본 점수) | 10 |

점수가 낮으면(특히 50 미만) 지금 수익 중이어도 카드에 `PRED⚠` 경고가 뜹니다 — 오류가 아니라 "근거가 식어가는 걸 미리 알려주는" 의도된 동작입니다.

### 4-4. 컷/등급 요약

| 구분 | 기준 |
|---|---|
| 관심(참고용) | 65 |
| 진입(동적) | 70(추세장) ~ 80(횡보장), 기본 75 |
| 매집·분산 | 80(고정) |
| Predict 등급 | 90+🔥 / 80+🟢 / 70+🟡 / 60+🟠 / 50+🔴 / 50미만⚫ |

카드 배경색: 🟢롱 🔴숏 🔵매집 🟣분산 🟡관심 (앱 안 "설명" 버튼에 전체 설명 있음)

## 5. 주요 기능

### 계좌/자금 흐름
- **외부통장**(거래소 밖 진짜 지갑)과 **거래소 잔고**가 분리되어 있습니다.
  - 통장입금/통장출금: 외부통장 자체의 입출금(입금은 새 돈, 출금은 실생활 지출 — 시스템 밖으로 사라짐)
  - 거래소충전/거래소출금: 외부통장 ↔ 거래소 간 이체
  - 화면에 순수익(누적 입출금 반영한 진짜 트레이딩 성과)도 같이 표시됨
- **크로스 마진 모드**(기본 켜짐, "크로스" 버튼으로 격리 마진과 전환 가능): 개별 포지션 손실이 -100%를 넘어도 계좌 총자산이 유지증거금(포지션 명목가치의 0.5%)을 버티면 청산 안 됨. 진입 시점에 여유현금이 유지증거금보다 적으면 애초에 진입 자체를 막습니다.
- "리셋" 버튼은 거래소 잔고 + 외부통장(누적 입출금 포함) 전부 초기화합니다.

### 화면 구성
- **코인 화면 / 포지션 화면**을 상단 탭으로 완전히 분리 (각각 전체 화면 사용, 둘 다 측면 스크롤바 있음)
- 코인 화면: 기준봉(1h/2h/6h/12h) 전환 버튼, 티커 검색(고정한 코인은 검색과 무관하게 항상 표시), 정렬 헤더
- 포지션 화면: 포지션별 Predict Score(P##▲▼), 청산가/증거금/마진비율 등 실시간 표시
- "리포트" 버튼: 현재 Long/Short Top10 지표 + 전체 지표 설명이 담긴 PDF를 서버 폴더에 생성 (fpdf2 필요)

### 서버 안전장치
- 강제청산은 1초 주기로 감시. 크로스 모드에서는 계좌가 감당 못 할 때 손실이 가장 큰 포지션부터 하나씩 정리(캐스케이드), 격리 모드에서는 포지션별 90% 손실 시 즉시 청산.
- 시뮬레이션 데이터(잔고/포지션)와 거래 기록은 각각 `simulation_data_usd.csv`, `trade_history_usd.csv`로 자동 저장·복구됩니다.

## 6. 데이터 수집 / 오프라인 분석

```bash
python data_collector.py
```
- `trading_server.py`의 `process_ticker()`를 그대로 재사용해 서버와 100% 동일한 로직으로 로그를 쌓습니다(계좌/포지션은 안 건드림).
- 3일마다 파일 자동 분리(`market_data_log_v3_YYYYMMDD_YYYYMMDD.csv`), 중간에 껐다 켜도 진행 중이던 구간에 이어서 씁니다.
- `screener_prepump.py market_data_log_v3_....csv`로 오프라인 스크리닝 가능(단, 옛 버전 함수 기준이라 최신 v3 배점과는 안 맞을 수 있음 — 필요시 갱신 요청).

## 7. 자주 겪는 문제

| 증상 | 원인/해결 |
|---|---|
| `ModuleNotFoundError: No module named 'numpy'`(또는 pandas, dateutil, dns, fpdf 등) | `requirements.txt` 기준으로 하나씩 pip install. Termux는 pandas/numpy를 pkg로 먼저 깔 것. |
| `pkg install tkinter` 안 됨 | 패키지명이 `python-tkinter`이고 `x11-repo`에 있음 (메인 저장소 아님). proot Ubuntu 안이면 `python3-tk`. |
| 클라이언트 켰는데 "서버: 연결 끊김" | 서버·클라이언트 둘 다 저장공간 권한(안드로이드 앱 권한) 확인. 서버가 실제로 돌고 있는지, 같은 폴더(`server_market.csv` 위치)를 보고 있는지 확인. |
| VNC/X11 환경에서 글자가 너무 작음 | 화면 폭만으로 모바일 여부를 판단하던 버그 수정됨(DPI도 같이 확인) — 최신 파일로 교체. |
| 한글 깨짐(네모/물음표) | Ubuntu 컨테이너에 한글 폰트+로케일 없어서. 위 2-2 항목 참고. |
| pip 설치 중 mirror 에러 | `termux-change-repo`로 미러 변경, 또는 잠시 후 재시도. Termux 전체가 `pkg update && pkg upgrade`로 버전 꼬임 정리 필요할 수도 있음. |

## 8. 알려진 한계 / 검증 안 된 부분 (중요)

- **이 점수 체계는 아직 실전 검증(백테스트)이 안 됐습니다.** 지금까지의 작업은 전부 "신호가 안 뜨던 문제"를 고치는 쪽이었지, "그 신호가 실제로 맞는지"는 확인 전입니다. `data_collector.py`로 데이터를 충분히(3일 이상) 모아서, 신호 발생 이후 실제 가격이 어떻게 움직였는지 검증하는 걸 강력히 권장합니다.
- OI 지속성(매집/분산 점수)은 진짜 며칠짜리 누적이 아니라 1시간 변화율로 근사합니다 — 장기 OI 히스토리 캐시는 아직 없습니다.
- CVD는 실제 체결 데이터가 아니라 OHLCV 기반 추정치입니다.
- "20일 박스"/"최근 3일" 같은 표현은 실제 일수가 아니라 **캔들 개수로 고정**되어 있어서, 기준봉(1h/2h/6h/12h)을 바꾸면 실제 기간도 같이 바뀝니다.
